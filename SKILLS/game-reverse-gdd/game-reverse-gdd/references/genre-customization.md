# 장르 감지 및 커스터마이징 가이드

> 게임의 장르를 자동 감지하고, 장르별 추가 섹션과 전용 차트를 역기획서에 포함하는 규칙을 정의한다.

---

## 장르 감지 로직

### 감지 프로세스

```
1. 게임명에서 키워드 매칭 (1차)
2. 시스템명에서 키워드 매칭 (2차)
3. 웹 리서치 결과에서 장르 정보 추출 (3차)
4. 복수 장르 해당 시 모두 기록
5. 감지 불확실 시 유저에게 AskUserQuestion으로 확인
```

### 감지 우선순위

1. **시스템명 키워드**: 가장 직접적인 단서 (최우선)
2. **게임명 키워드**: 게임 자체의 장르 힌트
3. **웹 리서치 결과**: 공식 분류, 스토어 태그, 커뮤니티 분류
4. **유저 확인**: 자동 감지가 불확실할 때 최종 확인

### 감지 신뢰도 기준

| 매칭 수 | 신뢰도 | 액션 |
|---------|--------|------|
| 키워드 3개 이상 매칭 | 높음 | 자동 확정, 유저에게 확인 표시만 |
| 키워드 1~2개 매칭 | 중간 | 유저에게 확인 질문 |
| 키워드 0개 매칭 | 낮음 | 웹 리서치 후 유저에게 선택지 제시 |

## 다중 장르 처리

### 다중 장르 판별 기준

```
1. 각 장르의 키워드 매칭 점수 계산
   - 1차 키워드 매칭: 3점
   - 2차 키워드 매칭: 2점
   - 3차 키워드 매칭: 1점

2. 점수 3점 이상인 장르를 모두 해당 장르로 기록

3. 해당 장르가 2개 이상이면 다중 장르로 처리
```

### 다중 장르 시 섹션 포함 규칙

| 규칙 | 설명 | 예시 |
|------|------|------|
| 전체 포함 | 해당하는 모든 장르의 추가 섹션을 포함 | RPG + 가챠 → 성장 밸런스 + 천장 시뮬레이션 |
| 우선순위 | 시스템과 가장 직접 관련된 장르의 섹션을 먼저 배치 | "가챠 시스템" 분석 → 가챠 섹션 우선, RPG 섹션 후순 |
| 중복 제거 | 유사한 차트가 있으면 통합 | 경제중심 + RPG의 재화 분석 → 통합 1개 섹션 |

### 일반적인 다중 장르 조합

| 조합 | 흔한 게임 유형 | 추가 섹션 |
|------|-------------|----------|
| RPG + 가챠 | 모바일 RPG | 성장 밸런스 + 천장 시뮬레이션 |
| RPG + 경제중심 | MMORPG | 성장 밸런스 + 경제 순환 심층 |
| 가챠 + 라이브서비스 | 모바일 수집형 | 천장 시뮬레이션 + 시즌 순환 |
| 전략 + 경제중심 | 전략 시뮬레이션 | 자원 균형 + 경제 순환 심층 |
| MOBA + 라이브서비스 | 경쟁 멀티플레이어 | 밸런스 메타 + 시즌 순환 |
| 방치형 + 가챠 | 방치형 RPG | 시간효율 + 천장 시뮬레이션 |
| RPG + 방치형 | 방치형 RPG | 성장 밸런스 + 시간효율 |
| 배틀로얄 + 라이브서비스 | 배틀로얄 라이브 | 전투 밀도 + 시즌 순환 |

## 유저 확인 프로세스

### AskUserQuestion 템플릿

장르 자동 감지 후 유저에게 확인할 때 사용하는 메시지 템플릿:

```
게임 "{게임명}"의 "{시스템명}" 시스템을 분석합니다.

감지된 장르: {장르1} (+ {장르2} ...)
매칭 키워드: {키워드1}, {키워드2}, ...

이 장르 분류가 맞나요?
1. 네, 맞습니다
2. 아니요, 다른 장르입니다 → (어떤 장르인지 알려주세요)
3. 추가 장르가 있습니다 → (추가할 장르를 알려주세요)
```

### 유저 응답 처리

| 유저 응답 | 처리 |
|----------|------|
| "네" / 확인 | 감지된 장르로 확정, 작업 시작 |
| 다른 장르 지정 | 유저 지정 장르로 변경, 추가 섹션 재설정 |
| 장르 추가 | 기존 장르 유지 + 추가 장르 포함 |
| 무응답/모름 | 감지된 장르로 진행 (기본값) |

장르를 전혀 감지하지 못했을 때 기본으로 적용하는 분석:

| 항목 | 기본값 |
