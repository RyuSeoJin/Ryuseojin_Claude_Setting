# 발표 자료 구성 가이드 (PPT / HTML)

> 역기획서를 발표 자료로 변환할 때의 구성 가이드.
> PPT: 4:3 비율 (1024x768) 기준, 15~20장.
> HTML: 단일 파일, 사이드바 TOC, Mermaid 렌더링 포함. → `assets/templates/html-output-spec.md` 참조

---

## 슬라이드 구성

| # | 슬라이드 | 내용 | 레이아웃 |
|---|---------|------|---------|
| 1 | 표지 | 게임명, 시스템명, 작성자, 날짜 | 중앙 정렬, 게임 로고/스크린샷 배경 |
| 2 | 목차 | 전체 슬라이드 구성 | 번호 목록 |
| 3 | 분석 개요 | 분석 대상, 범위, 목적 | 3단 카드 레이아웃 |
| 4-5 | 정의서 요약 | 시스템 정의, 용어, 범위, 관련 시스템 | 표 + 텍스트 |
| 6 | 설계 의도 | 5가지 관점 분석 결과 | 5개 아이콘 + 텍스트 |
| 7-8 | 구조도 | Mermaid 구조도 (렌더링 이미지) | 풀 페이지 다이어그램 + 범례 |
| 9-10 | 플로우차트 | 메인 플로우 + 서브 플로우 | 풀 페이지 다이어그램 |
| 11-13 | 상세 명세 | UI, 인터랙션, 애니메이션, 상태 전이 | 스크린샷 + 설명 분할 |
| 14-15 | 데이터 테이블 | 핵심 테이블 + ER 다이어그램 + 밸런싱 공식 | 표 + 그래프 |
| 16 | 예외 처리 | 핵심 엣지 케이스 요약 | 표 (상위 5개) |
| 17-18 | 비교 분석 | 비교 매트릭스 + 인사이트 | 비교 표 + 핵심 인사이트 카드 |
| 19 | 결론 & 개선안 | 핵심 발견, 개선 제안 | 불릿 리스트 |
| 20 | Q&A | 질문 유도 | 최소 디자인 |

## 디자인 가이드라인

| 용도 | 폰트 | 크기 | 굵기 |
|------|------|------|------|
| 슬라이드 제목 | Pretendard / Noto Sans KR | 28-32pt | Bold |
| 본문 | Pretendard / Noto Sans KR | 16-20pt | Regular |
| 표 내부 | Pretendard / Noto Sans KR | 14-16pt | Regular |
| 캡션/주석 | Pretendard / Noto Sans KR | 12pt | Light |
| 코드/수식 | D2Coding / Source Code Pro | 14pt | Regular |

### 레이아웃 규칙

1. **여백**: 상하좌우 최소 40px
2. **정렬**: 좌측 정렬 기본 (제목은 중앙 가능)
3. **한 슬라이드 최대 정보량**: 핵심 메시지 1개 + 보조 정보 3개
4. **표**: 최대 7행 × 5열 (초과 시 분할)
5. **다이어그램**: 풀 페이지 사용, 범례 포함
6. **스크린샷**: 좌측 60% 스크린샷 + 우측 40% 설명

### 슬라이드별 팁

- **표지**: 게임의 대표 스크린샷을 배경에 깔고 반투명 오버레이
- **다이어그램 슬라이드**: 텍스트 최소화, 다이어그램 중심
- **비교 분석**: 색상으로 게임 A/B를 구분
- **결론**: 핵심 인사이트 3개를 큰 폰트로 강조
